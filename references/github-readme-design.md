# GitHub README Design

这份设计稿只服务一个目标：
让 GitHub 访客在 5 秒内决定这项目值得继续看，甚至愿意点 Star 或转发。

不按官网做。
按 GitHub 项目的阅读习惯做。

## 目标

README 首屏要同时完成三件事：

1. 让人立刻知道这项目是干什么的
2. 让人觉得它有梗，值得分享
3. 让人发现它不是纯玩梗，是真的有用

## 首屏结构

推荐顺序不要改：

1. 项目名
2. 三句老师味对白
3. 4 个 badge
4. 一句最强定位语
5. 一句副标题
6. 一句反差句
7. 4 条短 bullet
8. 一个 4 到 6 行的对话预览

这个顺序的原因很简单：

- 先让人停下来
- 再让人笑一下
- 再让人知道用途
- 最后让人产生“这玩意儿真能用”的感觉

## 当前 README 顶部应该长什么样

### 第一层

`大学老师.skill`

### 第二层

三句对白：

- `这题我上课讲过。`
- `老师，您上课一共讲了 187 页 PPT。`
- `所以你们更应该自己总结。`

这三句的作用不是解释产品，是先把学生记忆打出来。

### 第三层

badge 建议保留 4 个就够：

- `MIT`
- `beta`
- `GitHub meme project`
- `期末复习 + 老师模拟`

badge 太多会显得像模板仓库。

### 第四层

主定位句必须大白话，最好一口气说清：

`把大学老师蒸馏成 AI Skill`

### 第五层

副标题负责解释输入和输出：

`导入 PPT、历年题、课堂录音和群聊，把任课老师变成一个会划重点、会答疑、会模拟出题风格的赛博老师。`

### 第六层

反差句一定要短，负责转发：

`你以为这是人格模拟，其实这是期末自救。`

### 第七层

短 bullet 只放 4 条：

- `老师说不考，我不信。`
- `PPT 不会帮你划重点，但老师.skill 会。`
- `一个比群公告更像人话的大学老师。`
- `把“课上讲过”做成一个真的能用的 Skill。`

## GitHub 封面图设计稿

如果你后面要补一张仓库头图，建议做 `1280 x 640`。

### 版式

左侧放大字，右侧放聊天截图风 mockup。

左侧结构：

1. 小标签：`期末周特供`
2. 大标题：`把大学老师蒸馏成 AI Skill`
3. 副标题：`会划重点，会答疑，会像老师本人一样回你`
4. 底部短句：`老师说不考，我不信。`

右侧 mockup 对话建议：

```text
你：老师，这个会考吗？

大学老师.skill：
我不直接回答这个问题。
你把作业、例题和上课重复次数放在一起看，
就知道它值不值得你今晚先看。
```

### 视觉关键词

- 米白底
- 深墨绿或深棕字
- 红笔批注感
- 课程 PPT 高亮框
- 聊天气泡像老师在群里回消息

不要做成：

- 泛 AI 科技蓝紫风
- 普通教育 SaaS
- 过于卡通的校园海报

它应该像：

- 被老师批改过的讲义
- 期末周流传的神秘复习资料
- 带一点互联网整活感的学习工具

## README 第二屏怎么排

第一屏之后，不要立刻讲技术实现。
应该先给三个东西：

1. 为什么这个项目会被转发
2. 一个很短的效果预览
3. 导入什么材料能得到什么结果

推荐顺序：

- `这项目为什么适合 GitHub 引流`
- `效果预览`
- `适合导入的材料`
- `项目目标`
- `使用方式`

## 最适合截图传播的内容

如果你想让别人转发，不是转 README 全文，而是转这几种内容：

### 1. 老师味对话

例子：

```text
你：老师，这一步为什么这么变？
老师.skill：定义写了吗？定义都不写，后面的正确率不归我管。
```

### 2. 期末冲刺清单

例子：

- `最后一晚只看这 5 个点`
- `这老师最爱扣分的 4 个地方`

### 3. “老师说不考”主题图

例子：

- `老师说不考，我把老师做成 skill 问了个清楚`

## 不要在首屏放什么

这些内容会拖慢 GitHub 转化：

- 太长的原理解释
- 太多 feature checklist
- 太完整的项目结构树
- 太多安装细节
- 一上来就贴大段 prompt

这些内容应该往下放。

## Star 转化句

适合放在 README 比较靠前的位置，但不要太像求赞：

- `如果你也被“这题上课讲过”伤害过，欢迎点个 Star。`
- `如果这玩意儿真救过你一次期末，可以给个 Star。`
- `如果你觉得这项目比群公告更像人话，点个 Star。`

## 推荐的 GitHub 社交标题

- `大学老师.skill：把“这题课上讲过”做成一个 AI 项目`
- `我把任课老师做成了一个会划重点的 Skill`
- `老师说不考，我决定把老师本人导进 AI`
- `不是角色扮演，是期末自救`

## 最后判断标准

README 顶部如果能让人立刻说出下面这句话，首屏就算成功：

`这项目有梗，而且我马上知道它能拿来干嘛。`
